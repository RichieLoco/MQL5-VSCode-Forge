# MQL5-VSCode-Forge

A streamlined MQL5 development workflow in Visual Studio Code for MetaTrader 5, replacing the need to use MetaEditor directly for compilation.

## Quick Start

### Prerequisites
- Visual Studio Code
- MetaTrader 5 installed (with metaeditor64.exe)
- Your MQL5 source files in `Experts/` and `Indicators/` directories

### Setup

1. **Clone/copy this project** to your workspace

2. **Find your MetaTrader Terminal ID**
   - Open Windows Explorer and paste this in the address bar:
   ```
   %APPDATA%\MetaQuotes\Terminal
   ```
   - You'll see a folder with a long ID (32 characters like `A1B2C3D4E5F60718293A4B5C6D7E8F90`)
   - Copy this ID

3. **Update config.json** (`.vscode/config.json`)
   - Open `.vscode/config.json` and paste your Terminal ID:
   ```json
   {
       "metaTraderPath": "C:\\Program Files\\MetaTrader 5\\metaeditor64.exe",
      "metaTraderTerminalId": "A1B2C3D4E5F60718293A4B5C6D7E8F90",
       "logFile": "compile.log",
       "sourceDirectories": ["Experts", "Indicators"]
   }
   ```
   - If MetaTrader 5 is installed in a non-standard location, update `metaTraderPath`

4. **Run setup script** (one time)
   - Open PowerShell in the workspace root
   - Run:
   ```powershell
   .\.vscode\setup-environment.ps1
   ```
   - This generates `c_cpp_properties.json` with your environment paths
   - If needed later, regenerate with: `.\.vscode\setup-environment.ps1 -Force`
   - **What the script does:**
     - Reads your Terminal ID from config.json
     - Builds the full include path: `%APPDATA%\MetaQuotes\Terminal\[YOUR_ID]\MQL5\Include`
     - Generates `.vscode\c_cpp_properties.json` with IntelliSense configuration
     - Enables C++ syntax highlighting and code completion for MQL5

5. **Add global keyboard shortcuts** (optional but recommended)
   - Open: File → Preferences → Keyboard Shortcuts (or press `Ctrl+K Ctrl+S`)
   - Click the document icon (top right) to edit keybindings.json
   - Add these entries:
   ```json
   {
       "key": "ctrl+shift+b",
       "command": "workbench.action.tasks.runTask",
       "args": "Compile MQL5 File"
   },
   {
       "key": "ctrl+shift+a",
       "command": "workbench.action.tasks.runTask",
       "args": "Compile All MQL5 Files"
   }
   ```

### Verification

After setup, verify everything works:
1. Open any `.mq5` file in the editor
2. Check that IntelliSense appears (you should see code completion and syntax highlighting)
3. Press `Ctrl+Shift+B` to compile the file
4. Check the terminal output - you should see compilation progress
5. Open `compile.log` to verify the compilation log was written

## Usage

### Compile Single File
1. **Open an .mq5 file** in the editor
2. **Run one of:**
   - Keyboard: `Ctrl+Shift+B` (or your custom shortcut)
   - Menu: Terminal → Run Task → "Compile MQL5 File"
   - Command Palette: `Ctrl+Shift+P` → Tasks: Run Task → "Compile MQL5 File"

**Output:** Compilation result appears in VS Code terminal and is appended to `compile.log`

### Compile All Files
1. **Run one of:**
   - Keyboard: `Ctrl+Shift+A` (or your custom shortcut)
   - Menu: Terminal → Run Task → "Compile All MQL5 Files"
   - Command Palette: `Ctrl+Shift+P` → Tasks: Run Task → "Compile All MQL5 Files"

**Output:** 
- All .mq5 files in `Experts/` and `Indicators/` are compiled
- Results appended to `compile.log`
- Summary shows: Total files, Passed, Failed
- Failed files listed with error details

## File Structure

```
.vscode/
  ├── config.json                    # Environment configuration (CUSTOMIZE THIS)
  ├── setup-environment.ps1          # Run once to generate c_cpp_properties.json
  ├── settings.json                  # Workspace settings (file associations for .mq5/.mqh)
  ├── keybindings.json               # Suggested keyboard shortcuts for compile tasks
  ├── c_cpp_properties.json          # Generated by setup-environment.ps1 (AUTO-GENERATED, not committed)
  ├── tasks.json                     # VS Code build tasks
  ├── compile-active-mql5.ps1        # PowerShell script: compile single file
  └── compile-all-mql5.ps1           # PowerShell script: compile all files

Experts/                              # Your Expert Advisor .mq5 files
Indicators/                           # Your Indicator .mq5 files
compile.log                           # Compilation output log (git-ignored)
```

## Configuration

### config.json

All environment-specific settings are in `.vscode/config.json`:

| Setting | Example | Purpose |
|---------|---------|---------|
| `metaTraderPath` | `C:\Program Files\MetaTrader 5\metaeditor64.exe` | Path to MetaEditor executable |
| `metaTraderTerminalId` | `A1B2C3D4E5F60718293A4B5C6D7E8F90` | Your MT5 Terminal ID (from %APPDATA%\MetaQuotes\Terminal) |
| `logFile` | `compile.log` | Output log filename (relative to workspace root) |
| `sourceDirectories` | `["Experts", "Indicators"]` | Directories to scan for .mq5 files |

**Share this file with your team** - they only need to update their Terminal ID and MetaTrader path if different.

### c_cpp_properties.json

*Auto-generated by `setup-environment.ps1`*

This file is generated once during setup to configure VS Code's C/C++ extension with your MQL5 include paths. It's **not shared** with the team (in .gitignore) so each developer can have their own paths.

## Sharing with Others

To share this setup with teammates:

1. **They copy the `.vscode/` folder** to their project
2. **They update `.vscode/config.json`:**
   - Paste their MetaTrader Terminal ID
   - Update MetaTrader path if different
3. **They run the setup script:**
   ```powershell
   .\.vscode\setup-environment.ps1
   ```
4. **Done!** They can now use the same compilation workflow

The `.gitignore` file already excludes:
- `compile.log` (build artifacts)
- `*.ex5` (compiled binaries)  
- `c_cpp_properties.json` (auto-generated, environment-specific)

## Transitioning from MetaEditor

### MetaEditor Habits → VS Code Equivalents

| MetaEditor | VS Code |
|-----------|---------|
| Press Compile button | Press `Ctrl+Shift+B` |
| Compile All | Press `Ctrl+Shift+A` |
| View compile.log | Open file in editor or check Terminal output |
| Keep editor open | Use VS Code instead |

### Key Differences

| Aspect | MetaEditor | VS Code |
|--------|-----------|---------|
| **Compilation** | Built-in, single UI | Task-based, scriptable |
| **Editing** | Limited syntax highlighting | Full IntelliSense (with setup) |
| **Terminal** | Embedded | Integrated VS Code terminal |
| **Version Control** | None | Full Git support |
| **Extensions** | None | Large ecosystem |

## Troubleshooting

### "config.json not found" error
- Ensure `.vscode/config.json` exists in your workspace root
- The scripts must be run from the correct workspace

### "metaTraderTerminalId not found" when running setup script
- Run `setup-environment.ps1` and check the error message
- Manually find your Terminal ID in Windows Explorer at: `%APPDATA%\MetaQuotes\Terminal`
- Update `config.json` with the correct ID

### Compilation fails with "MetaEditor not found" 
- Check the path in `.vscode/config.json` points to your actual `metaeditor64.exe`
- Verify MetaTrader 5 is installed at that location

### IntelliSense not working for MQL5
- Run `setup-environment.ps1` again with `-Force` flag
- Restart VS Code
- Verify your Terminal ID in `config.json` is correct

### Shortcuts not working
- **Option 1:** Use the Command Palette (`Ctrl+Shift+P`) → Tasks: Run Task
- **Option 2:** Edit global keybindings at `%APPDATA%\Code\User\keybindings.json`

### Compilation succeeds but no output appears
- Check the `compile.log` file in your workspace root
- Compilation log is appended to this file every run
